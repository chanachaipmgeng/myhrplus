Complete guide to all loading components: Local Loading, Global Loading, Basic Spinner, and Skeleton Loading. Choose the right component for your use case.

1. Local Loading (LoadingComponent)

When to Use Which?

LoadingComponent (Local Loading)

‚úÖ Use for component-level loading states
‚úÖ When you need to show loading within a specific container
‚úÖ For forms, cards, or sections that load independently

LoadingSpinnerComponent (Global Loading)

‚úÖ Use for application-level loading states
‚úÖ When multiple components need to show loading simultaneously
‚úÖ For API calls that affect the entire application
‚úÖ Add once in app.component.html or main layout

SpinnerComponent (Basic Spinner)

‚úÖ Use for simple, standalone spinner needs
‚úÖ When you need custom size, color, or full-screen overlay
‚úÖ For buttons, inline loading, or custom loading states

SkeletonLoaderComponent (Skeleton Loading)

‚úÖ Use for content placeholders while loading
‚úÖ When you want to show the structure of content that will appear
‚úÖ For data tables, lists, cards, or any structured content
‚úÖ Improves perceived performance
